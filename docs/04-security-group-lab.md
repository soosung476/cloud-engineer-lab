# Security Group 점검 및 실습

## 1. 실습 목적
Security Group의 인바운드 규칙이 SSH/HTTP 접속에 어떤 영향을 주는지 확인한다.

## 2. 실습 환경
- AWS EC2 Ubuntu
- Security Group
- SSH / HTTP

## 3. 실습 목표
- 22 포트 의미 이해
- 80 포트 의미 이해
- 인바운드 규칙 변경 실습
- 접속 허용/차단 변화 확인

## 4. 작업 과정
- 현재 보안 그룹 규칙 확인
- 80 포트 허용 상태에서 웹 접속 확인
- 80 포트 제거 후 접속 실패 확인
- 80 포트 다시 추가 후 접속 복구 확인
- 22 포트를 내 IP로 제한

## 5. 확인 결과
- Security Group 규칙에 따라 외부 접속 가능 여부가 즉시 달라짐을 확인
- 서비스가 살아 있어도 SG가 막으면 접속 불가함을 확인

## 6. 배운 점
- Security Group은 클라우드 레벨 방화벽 역할을 한다
- 운영에서 접속 장애는 서비스 상태만이 아니라 네트워크 정책도 함께 봐야 한다
