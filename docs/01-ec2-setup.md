# EC2 생성 및 초기 접속

## 1. 실습 목적

AWS EC2에 Ubuntu 서버를 생성하고,  
SSH를 통해 원격 접속이 가능한 상태까지 구성한다.

이 실습의 목적은 단순히 인스턴스를 만드는 것이 아니라,  
**클라우드에서 리눅스 서버를 하나 준비하고 접속 가능한 운영 상태로 만드는 기본 흐름**을 익히는 데 있다.

---

## 2. 실습 환경

- Cloud: AWS EC2
- OS: Ubuntu
- Instance Type: t3.micro
- Local Environment: MacBook M1
- Access Method: SSH
- Version Control: GitHub (`cloud-engineer-lab`)

---

## 3. 구성 내용

이번 실습에서는 아래와 같은 기본 구성을 사용했다.

- EC2 Ubuntu 인스턴스 1대 생성
- 퍼블릭 IP를 통해 외부에서 접속
- Security Group으로 SSH / HTTP 허용
- 키 페어를 이용한 SSH 인증
- 이후 nginx 설치를 위한 기본 서버 준비 완료

---

## 4. 주요 설정

### 인스턴스
- Ubuntu 기반 EC2 인스턴스 생성
- 프리티어 범위에서 실습 가능한 타입 사용

### 보안 그룹
- SSH (22): 접속용
- HTTP (80): 웹 서버 테스트용

### 인증 방식
- 키 페어 기반 SSH 접속 사용

---

## 5. 작업 과정

### 5-1. EC2 인스턴스 생성
AWS 콘솔에서 Ubuntu EC2 인스턴스를 생성했다.

확인한 항목:
- 인스턴스 타입
- 키 페어 연결
- 보안 그룹 설정
- 퍼블릭 IP 할당 여부

### 5-2. 보안 그룹 확인
원격 접속과 웹 접속 테스트를 위해 필요한 포트를 열었다.

- 22/tcp: SSH 접속
- 80/tcp: HTTP 접속

이 단계에서 보안 그룹은  
**클라우드 레벨에서 인바운드 트래픽을 제어하는 첫 번째 보안 장치**라는 점을 함께 이해했다.

### 5-3. SSH 접속
로컬 Mac 터미널에서 SSH로 접속을 시도했다.

접속이 성공하면 아래를 확인할 수 있다.
- 원격 리눅스 셸 진입
- 현재 사용자 확인
- 서버 내부 명령어 실행 가능

---

## 6. 확인 결과

이번 실습에서 아래 항목을 확인했다.

- EC2 인스턴스 생성 성공
- Running 상태 확인
- 퍼블릭 IP 확인
- SSH 접속 성공
- 이후 nginx 설치를 진행할 수 있는 기본 상태 확보

즉,  
**AWS 위에 리눅스 서버 1대를 실제로 준비하고 원격 관리 가능한 상태로 만들었다.**

---

## 7. 운영 관점에서 중요한 이유

클라우드 운영 / 시스템 엔지니어 입장에서 EC2 생성은 단순한 클릭 작업이 아니다.

이 단계에서 실제로 중요한 것은 다음이다.

- 서버를 외부에서 접근 가능하게 만드는 방법 이해
- 네트워크와 보안 설정이 접속 가능 여부에 직접 영향을 준다는 점 이해
- SSH를 통해 원격 서버를 관리하는 기본 흐름 숙지
- “서버가 있다”와 “운영 가능한 상태다”는 다르다는 점 이해

즉,  
이 실습은 이후의 nginx, 모니터링, 로드밸런싱, 오토스케일링 실습의 출발점이다.

---

## 8. 발생할 수 있는 문제

EC2를 생성했더라도 실제 접속 단계에서는 여러 문제가 발생할 수 있다.

### SSH 접속 실패 가능 원인
- 인스턴스가 Running 상태가 아님
- 퍼블릭 IP 확인 오류
- 보안 그룹에서 22 포트 미허용
- 키 파일 권한 문제
- 잘못된 사용자명 사용
- 네트워크 환경 문제

이 실습을 통해  
**서버 접속 문제는 단일 원인이 아니라 인스턴스 상태, 보안, 인증, 네트워크가 함께 맞아야 해결된다**는 점을 배웠다.

---

## 9. 트러블슈팅 체크리스트

SSH 접속이 되지 않을 때는 아래 순서로 확인한다.

1. EC2 인스턴스가 Running 상태인지 확인
2. 퍼블릭 IP 또는 퍼블릭 DNS가 맞는지 확인
3. Security Group에서 22 포트가 열려 있는지 확인
4. 사용 중인 키 파일이 맞는지 확인
5. 키 파일 권한이 적절한지 확인
6. 사용자명이 Ubuntu인지 확인
7. 로컬 터미널 명령어 입력이 올바른지 확인

---

## 10. 배운 점

- EC2 생성 자체보다 “접속 가능한 상태”를 만드는 과정이 더 중요하다.
- 보안 그룹은 서버 접속 가능 여부에 직접적인 영향을 준다.
- SSH 접속은 클라우드 운영의 가장 기본적인 시작점이다.
- 이후 실습은 모두 이 서버에 접속할 수 있다는 전제 위에서 진행된다.
